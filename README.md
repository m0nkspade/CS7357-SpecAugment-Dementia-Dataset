# CS7357 — SpecAugment on Dementia Speech Dataset

Binary speech classification (dementia vs. nodementia) using SpecAugment data
augmentation and fine-tuned `facebook/wav2vec2-xls-r-300m`.

---

## Pipeline

| Notebook | Purpose |
|----------|---------|
| `01-prepare-augmented-data.ipynb` | Scan audio dirs, stratified split, apply SpecAugment, save `train_dm.csv` / `valid_dm.csv` |
| `02-finetune.ipynb` | Fine-tune Wav2Vec2 on augmented training set with class-weighted loss, push to HuggingFace Hub |
| `03-eval.ipynb` | Load fine-tuned model, run inference on validation set, classification report + confusion matrix |

## SpecAugment Configuration (SM — Switchboard Mild)

| Param | Value | Meaning |
|-------|-------|---------|
| F | 15 | Max frequency mask width (STFT bins) |
| mF | 2 | Number of frequency masks |
| T | 70 | Max time mask width (STFT frames) |
| p | 0.2 | Max proportion of frames masked |
| mT | 2 | Number of time masks |
| copies | dementia: 3, nodementia: 2 | Augmented copies per original (more for minority class) |

---

## Key Design Decisions

- **SpecAugment in STFT domain** — masks are applied to the complex spectrogram then inverted back to audio via iSTFT, so augmented files are standard `.wav` files loadable with `torchaudio`
- **Augmented files never enter validation** — the train/valid split is performed on originals only; augmented copies are appended to the training set afterward
- **Class-weighted loss** — inverse-frequency weights passed to `CrossEntropyLoss` to counteract the dementia/nodementia imbalance
- **Preprocessing cache** — processed tensors saved to Google Drive via `save_to_disk` so repeated runs skip the `.map()` pass; controlled by `CLEAR_CACHE` flag in notebook 02
